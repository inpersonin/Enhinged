01 — MODEL ARCHITECTURE

Architecture

GPT-2 style nanoGPT stack with token and position embeddings, masked multi-head attention, feed-forward blocks, residual connections, and tied input/output embeddings.

  • block_size = 256
  • n_layer = 6
  • n_head = 6
  • n_embd = 384

Training Snapshot

The checkpoint matches the metrics file and represents the best validation point from the 5000-iteration run.

  • best_val_loss = 1.4363
  • best_val_perplexity = 4.2
  • checkpoint = checkpoints/best.pt
  • dataset = Hinglish-Everyday-Conversations-1M

Model Flow

user → tokens → transformer → reply

Prompt enters as GPT-2 BPE tokens, moves through the transformer blocks, and the decoder samples the next assistant token autoregressively.

Serving Flow

GitHub Pages + HF Space

GitHub Pages hosts the static site. The HF Space FastAPI backend loads the checkpoint and answers the `/generate` request.

02 — WORKFLOW

Inference Path

User prompt GPT-2 BPE 256-token context Transformer blocks Assistant reply

Conversation history uses the same speaker labels as the corpus so the model sees the training pattern it learned on.

Backend Resilience

The API now lazy-loads the checkpoint, exposes startup error state in `/health`, and avoids hard failure if the model does not come up immediately.

  • /health reports startup error state
  • /generate loads on demand if needed
  • /load can swap checkpoints
  • CORS is open for GitHub Pages
